flowchart TD

## Level 1 — تفاصيل العمليات الداخلية

```mermaid
flowchart TD
    CUSTOMER["👤 العميل"]
    EMPLOYEE["👷 الموظف"]
    MANAGER["🛠️ المدير"]

    P1(["P1\nإدارة العملاء\nوالسيارات"])
    P2(["P2\nإدارة الحجوزات"])
    P3(["P3\nمعالجة المدفوعات"])
    P4(["P4\nالتقارير والإحصائيات"])

    D1[("🗄️ D1\nالعملاء")]
    D2[("🗄️ D2\nالحجوزات")]
    D3[("🗄️ D3\nالخدمات")]
    D4[("🗄️ D4\nالمدفوعات")]

    CUSTOMER -- "بيانات شخصية + سيارة" --> P1
    P1 -- "تخزين / تحديث" --> D1
    D1 -- "بيانات العميل" --> P1

    CUSTOMER -- "طلب حجز" --> P2
    P1 -- "بيانات العميل والسيارة" --> P2
    P2 -- "تخزين الحجز" --> D2
    D2 -- "تفاصيل الحجز" --> P2
    D3 -- "قائمة الخدمات والأسعار" --> P2
    EMPLOYEE -- "تحديث حالة الحجز" --> P2
    P2 -- "الحجز المؤكد" --> P3

    CUSTOMER -- "الدفع" --> P3
    P3 -- "تخزين الدفعة" --> D4
    D4 -- "سجل المدفوعات" --> P3
    P3 -- "فاتورة" --> CUSTOMER

    D2 -- "بيانات الحجوزات" --> P4
    D4 -- "بيانات المدفوعات" --> P4
    P4 -- "تقارير وإحصائيات" --> MANAGER

    MANAGER -- "إضافة / تعديل خدمات" --> D3
    MANAGER -- "إدارة الموظفين" --> P1
```

---

## شرح العمليات

| العملية | المسؤول | الوصف |
|---------|---------|--------|
| **P1** إدارة العملاء والسيارات | أنت (فرونت اند) + يوسف (API) | إضافة وتعديل بيانات العملاء وسياراتهم |
| **P2** إدارة الحجوزات | أنت + يوسف | حجز جديد، تحديث الحالة، الإلغاء |
| **P3** معالجة المدفوعات | أنت + يوسف | تسجيل الدفع وإصدار الفاتورة |
| **P4** التقارير | أنت (عرض) + يوسف (بيانات) | إحصائيات وتقارير دورية للمدير |

## مصادر البيانات

| المصدر | النوع | الجدول في قاعدة البيانات |
|--------|-------|--------------------------|
| D1 | قاعدة بيانات | `clients` + `cars` |
| D2 | قاعدة بيانات | `bookings` |
| D3 | قاعدة بيانات | `services` |
| D4 | قاعدة بيانات | `payments` |
